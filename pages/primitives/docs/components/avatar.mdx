  - Automatic and manual control over when the image renders.
  - Fallback part accepts any children.

### Installation

Install the component from your command line.

```bash
npm install @radix-ui/react-avatar
```

### Anatomy

Import the components and piece the parts together.

```jsx
import { Root, Image, Fallback } from '@radix-ui/react-avatar';

export default () => (
  <Avatar>
    <AvatarImage />
    <AvatarFallback />
  </Avatar>
);
```

### Basic example

Create your styled avatar component from the primitive parts.

**Note:** This example uses [Stitches](https://stitches.dev) but any styling solution is [compatible](../overview/styling).

```jsx live manual
// import { styled } from 'path-to/stitches.config';
import * as AvatarPrimitive from '@radix-ui/react-avatar';

const Avatar = styled(AvatarPrimitive.Root, {
  display: 'inline-flex',
  alignItems: 'center',
  justifyContent: 'center',
  verticalAlign: 'middle',
  overflow: 'hidden',
  userSelect: 'none',
  width: 48,
  height: 48,
  borderRadius: 24,
});

const Image = styled(AvatarPrimitive.Image, {
  width: '100%',
  height: '100%',
  objectFit: 'cover',
});

const Fallback = styled(AvatarPrimitive.Fallback, {
  width: '100%',
  height: '100%',
  display: 'flex',
  alignItems: 'center',
  justifyContent: 'center',
  backgroundColor: 'dodgerblue',
});

export default () => (
  <Avatar>
    <Image src="https://picsum.photos/id/1005/400/400" />
    <Fallback>UI</Fallback>
  </Avatar>
);
```

### API Reference

#### Avatar

`Avatar` is the root component.

#### Image

`AvatarImage` is the image to render. By default `AvatarImage` will only render when it has actually been loaded. You can use the `onLoadingStatusChange` handler if you need finer control.

#### Fallback

`AvatarFallback` provides a fallback element that will only render anytime `AvatarImage` hasn't loaded. This means whilst it's loading, or if there was an error. If you don't want it to flash quickly whilst it's loading, you can provide a `delayMs` prop which will delay its rendering a bit. Otherwise, if you need finer control you can use the `onLoadingStatusChange` handler on `AvatarImage`.

#### Clickable Avatar with tooltip

```jsx live manual compact
import * as AvatarPrimitive from '@radix-ui/react-avatar';
import * as TooltipPrimitive from '@radix-ui/react-tooltip';

const Avatar = styled(AvatarPrimitive.Root, {
  display: 'inline-flex',
  alignItems: 'center',
  justifyContent: 'center',
  verticalAlign: 'middle',
  overflow: 'hidden',
  userSelect: 'none',
  width: 48,
  height: 48,
  borderRadius: 24,
  appearance: 'none',
  border: 'none',
  padding: 0,
});

const Image = styled(AvatarPrimitive.Image, {
  width: '100%',
  height: '100%',
  objectFit: 'cover',
});

const Fallback = styled(AvatarPrimitive.Fallback, {
  width: '100%',
  height: '100%',
  display: 'flex',
  alignItems: 'center',
  justifyContent: 'center',
  backgroundColor: 'dodgerblue',
});

const TooltipTrigger = styled(TooltipPrimitive.Trigger, {
  backgroundColor: 'transparent',
  border: 'none',
  padding: 0,
});

const TooltipContent = styled(TooltipPrimitive.Content, {
  borderRadius: 3,
  padding: '5px 10px',
  fontSize: 14,
  backgroundColor: 'gainsboro',
  color: 'black',
});

export default () => (
  <TooltipPrimitive.Root>
    <TooltipTrigger>
      <Avatar onClick={() => alert('Clicked!')}>
        <Image src="https://picsum.photos/id/1006/400/400" />
        <Fallback>AB</Fallback>
      </Avatar>
    </TooltipTrigger>

    <TooltipContent side="top">
      Tooltip content
      <TooltipPrimitive.Arrow style={{ fill: 'gainsboro' }} />
    </TooltipContent>
  </TooltipPrimitive.Root>
);
```
