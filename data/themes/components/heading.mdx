## API Reference

This component is based on the `h1` element and supports common margin props `m`, `mx`, `my`, `ml`, `mr`, `mt`, `mb`.

<ThemesPropsTable name="heading" />

## Examples

### As another element

Use the `asChild` prop to change the heading level. This prop is purely semantic and does not change the visual appearance .

```jsx live=true
<Heading>Level 1</Heading>
<Heading asChild>
  <h2>Level 2</h2>
</Heading>
<Heading asChild>
  <h3>Level 3</h3>
</Heading>
```

### Size

Use the `size` prop to control the size of the heading. The prop also provides correct line height and corrective letter spacing—as text size increases, the relative line height and letter spacing decreate.

You can change the size across breakpoints using the object syntax.

> Heading sizes match [Text](/docs/themes/components/text) sizes. However, the line heights and letter spacing are set tighter.

```jsx live="true"
<Flex direction="column" gap="3" width="max-content">
  <Heading size="9">Portez ce vieux whisky</Heading>
  <Heading size="8">Portez ce vieux whisky</Heading>
  <Heading size="7">Portez ce vieux whisky</Heading>
  <Heading size="6">Portez ce vieux whisky</Heading>
  <Heading size="5">Portez ce vieux whisky</Heading>
  <Heading size="4">Portez ce vieux whisky</Heading>
  <Heading size="3">Portez ce vieux whisky</Heading>
  <Heading size="2">Portez ce vieux whisky</Heading>
  <Heading size="1">Portez ce vieux whisky</Heading>
</Flex>
```

### Trim

Use the `trim` prop to trim the leading space at the start, end, or both sides of the text box. You can change the leading trim across breakpoints using the object syntax.

The prop works similarly to the upcoming [half-leading control](https://www.w3.org/TR/css-inline-3/#leading-trim) spec, but uses a common [negative margin workaround](https://seek-oss.github.io/capsize/) under the hood for cross-browser support.

```jsx live="true"
<Flex direction="column" gap="3">
  <Heading
    trim="normal"
    style={{
      background: 'var(--monoA2)',
      borderTop: '1px dashed var(--monoA7)',
      borderBottom: '1px dashed var(--monoA7)',
    }}
  >
    Without trim
  </Heading>
  <Heading
    trim="both"
    style={{
      background: 'var(--monoA2)',
      borderTop: '1px dashed var(--monoA7)',
      borderBottom: '1px dashed var(--monoA7)',
    }}
  >
    With trim
  </Heading>
</Flex>
```

Trimming the leading is useful when dialing in vertical spacing in cards or other “boxy” components. Otherwise, padding looks larger on top and bottom than on the sides.

```jsx live="true"
<Flex direction="column" gap="3">
  <Box
    style={{
      background: 'var(--monoA2)',
      border: '1px dashed var(--monoA7)',
    }}
    p="4"
  >
    <Heading mb="1" size="3">
      Without trim
    </Heading>
    <Text>
      The goal of typography is to relate font size, line height, and line width
      in a proportional way that maximizes beauty and makes reading easier and
      more pleasant.
    </Text>
  </Box>

  <Box
    p="4"
    style={{
      background: 'var(--monoA2)',
      border: '1px dashed var(--monoA7)',
    }}
  >
    <Heading mb="1" size="3" trim="start">
      With trim
    </Heading>
    <Text trim="end">
      The goal of typography is to relate font size, line height, and line width
      in a proportional way that maximizes beauty and makes reading easier and
      more pleasant.
    </Text>
  </Box>
</Flex>
```

### Color

Use the `color` prop to assign a specific color value, ignoring the global theme.

```jsx live=true
<Flex gap="3">
  <Heading color="crimson">Crimson</Heading>
  <Heading color="indigo">Indigo</Heading>
  <Heading color="grass">Grass</Heading>
  <Heading color="orange">Orange</Heading>
</Flex>
```
